# Agent for policy-driven execution of enterprise operations

***Applies to***: ***Dynamics 365 Commerce***

This template creates an agent with a user-defined and configurable set of processes and policies. After validating a user's prompt against your policies, the agent uses your process definitions to plan and run tasks in Dynamics 365 by using the Model Context Protocol (MCP) server for finance and operations apps.
All of the agent's actions occur in the user's context and use the Dynamics 365 role-based security, approvals, and auditing.

> [!NOTE]
> The agent templates aren't yet available for download. If you're interested in the commerce agent templates, [fill in this form](https://aka.ms/commerce-agent-templates).

## Overview

The agent automates day-to-day tasks in finance and operations apps. The agent runs customer-defined processes and policies with auditable and controlled autonomy, which reduces the manual effort and improves consistency and compliance.

## Key benefits

- Policy-driven, model-first execution with configurable guardrails​
- Customer-maintained catalog of approved executable processes​
- Tool-based integration via the MCP server​
- Supports approvals and exception handling for higher-risk actions​
- Operational logging and traceability for audit and governance

## Scenarios

- Execute routine operational playbooks, such as credit holds, replenishment, and invoice exception triage.
- Monitor for policy breaches and raise tasks for resolution.
- Run scheduled controls, like period-end checks, and summarize results.
- Assist operational teams in Teams with guided, policy-aware actions.
- Orchestrate multistep ERP processes with human approval where required.
